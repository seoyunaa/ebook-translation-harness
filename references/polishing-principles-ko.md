# 전자책 통합 윤문 원칙

이 문서는 초벌 번역 `draft_ko`를 독자용 `polished_ko`로 다듬을 때 적용하는 범용 원칙입니다.
윤문은 새로운 번역이나 요약이 아니라 **의미와 서지 구조를 보존하는 문장 층 교정**입니다.

이 원칙은 다음 공개 저장소의 서로 다른 장점을 전자책 작업에 맞게 합성했습니다.

- [`translate-book-to-korean`](https://github.com/seoyunaa/translate-book-to-korean)의 번역 플레이북: 단락 단위 번역, 자기비평, 번역투 교정, 용어집 일관성
- [`im-not-ai`](https://github.com/seoyunaa/im-not-ai)의 `humanize-korean`: 번역투와 반복적인 AI 문체 패턴 탐지, 과윤문 방지

여기에 학술서 검토에서 확인된 레지스터, 확신도, 인용·사료·각주 보존 원칙을 범용 규칙으로 보강했습니다. 특정 저자 문체, 논문별 용어, 사료 예문이나 비공개 작업 정보는 가져오지 않습니다.

공개 출처 검토 기준 버전(2026-07-15): `translate-book-to-korean` `8169fb9`, `im-not-ai` `8071726`.

## 충돌 시 우선순위

1. 원문 내용과 전자책 구조 보존
2. 책별 용어집·문체표와 사용자의 명시적 선택
3. 학술 인용·확신도·장르 레지스터 보존
4. 번역투 교정
5. 일반적인 AI 문체 교정

따라서 일반 글에서 권하는 목록·헤딩 제거는 전자책에 적용하지 않습니다. 헤징이나 명사화도 학술적 기능이 있으면 유지하며, 특정 저장소의 기본 존댓말보다 원문의 장르와 사용자가 선택한 문체가 우선합니다.

## 1. 입력과 출력

- 입력: 같은 작업 ID의 `draft_ko/<task-id>.md`
- 출력: `polished_ko/<task-id>.md`
- 원문 근거: 필요할 때만 `source/<task-id>.md`와 대조
- 금지: `source`와 `draft_ko` 덮어쓰기, 요약, 내용 추가, 장·절 재배열

초벌 번역이 없으면 윤문을 시작하지 않습니다. 의미가 불분명하면 추측해서 고치지 말고 해당 표현을 유지한 뒤 검토 필요 사항으로 보고합니다.

번역을 시작하기 전에는 핵심 용어, 인명·지명, 호칭, 첫 등장 원어 병기와 문체를 `03_outputs/translations/assets/<book_id>/glossary_ko.md` 하나에 확정합니다. 병렬 작업자는 한 배치가 도는 동안 이 파일을 읽기 전용으로 사용합니다. 새 용어나 충돌은 각자 보고하고, 배치 사이에 한 명의 조정자만 합의된 변경을 병합한 뒤 다음 배치를 시작합니다.

## 2. 절대 보존 항목

다음 요소는 표현을 자연스럽게 만든다는 이유로 바꾸거나 누락하지 않습니다.

1. 장·절 제목의 수준, 순서, 번호와 본문 위치
2. 인명, 지명, 기관명, 관직명, 모델명과 첫 등장 원어·한자 병기
3. 수치, 날짜, 연대, 단위, 통계식과 수학·화학 표기
4. 직접 인용, 사료 블록, 외국어 전사와 인용부호 안의 문구
5. 각주 번호, 각주 본문, 참고문헌 정보, 권·호·면수와 URL
6. 표의 행·열 구조와 셀 대응, 그림·표 번호와 캡션
7. 코드, 식별자, 파일명처럼 문자 그대로 보존해야 하는 표기
8. 원문의 확실성 수준: 단정, 추정, 유보의 차이

## 3. 번역투 교정

문맥을 확인한 뒤 해당하는 부분만 보수적으로 고칩니다.

- `~에 대해`, `~에 있어서`, `~와 관련하여`를 알맞은 조사나 구체 동사로 줄입니다.
- `~에 의해` 피동은 행위자가 분명할 때 능동으로 바꿉니다. 실제 피동은 유지합니다.
- `~되어지다` 같은 이중 피동과 불필요한 `~지게 되다`를 단일 피동·능동으로 고칩니다.
- `가지고 있다`, `진행을 하다` 같은 영어식 경동사 구문을 자연스러운 본동사로 바꿉니다.
- 영어 대명사를 반복한 `그·그녀·그것·그들`은 문맥상 가능한 곳에서 생략하거나 명칭으로 되돌립니다.
- 복수가 이미 분명한데 붙은 불필요한 `-들`은 문맥을 확인해 줄입니다.
- 영어의 `A of B`를 그대로 옮긴 어색한 `B의 A`는 한국어 합성어·조사·절로 자연스럽게 풉니다.
- 세 겹 이상 겹친 관형구는 문장을 나누거나 뒤 절로 풉니다.
- `~에서의`, `~으로의`, `~에로의` 같은 이중 조사는 절이나 간단한 구로 풉니다.
- 영어 진행형을 옮긴 `~고 있다`는 진행 의미가 없을 때 단순 시제로 바꿉니다.
- `그리고·그래서·왜냐하면`을 문장마다 대응시킨 경우 연결어미·문장 분할·생략으로 조정합니다.
- 명사화가 겹쳐 뜻이 흐려진 문장은 동사·형용사 중심으로 고칩니다. 학술 개념어는 유지합니다.

## 4. AI 문체 교정

다음 패턴이 반복될 때만 줄이거나 변주합니다. 한 번 쓰였다는 이유로 기계적으로 삭제하지 않습니다.

- 문두의 `또한·따라서·즉·나아가·아울러`가 매 문장을 이끄는 패턴
- `결론적으로`, `시사하는 바가 크다`, `주목할 만하다`처럼 구체 내용이 없는 결산 문구
- `A가 아니라 B다`, `X에서 Y로`, 세 항목 병렬처럼 같은 틀이 연속되는 문장
- 과장 형용사, 추상 주어의 의인화, 원문에 없던 비유와 수사
- 문장 길이와 종결어미가 지나치게 균일한 리듬
- 불필요한 굵은 글씨, 이모지, 콜론 부제, 목록화
- 같은 주장을 첫 문장과 마지막 문장에서 되풀이하는 문단

## 5. 학술서 예외

일반 문체 교정 규칙을 학술문에 기계적으로 적용하지 않습니다.

- 실제 불확실성을 나타내는 `~로 보인다`, `~일 수 있다`는 정당한 헤징이므로 유지합니다.
- `그렇다면`, `그런데`, `요컨대`, `주목할 것은`은 논증의 큰 매듭에서 쓸 수 있습니다.
- 학술 한자어와 필요한 명사화는 쉬운 말로 무조건 치환하지 않습니다.
- `이를 통해 ~을 알 수 있다`는 사료 분석의 정형일 수 있으므로 문맥을 보고 판단합니다.
- 저자나 책에 별도 용어표·문체표가 있으면 이 범용 원칙보다 우선합니다.

## 6. 기본 실행 모드

### Fast

8,000자 이하의 일반 조각은 한 번의 윤문과 자체 검증으로 처리합니다.

1. 절대 보존 항목을 먼저 잠급니다.
2. 번역투를 교정합니다.
3. 반복되는 AI 문체만 교정합니다.
4. 초벌 번역과 대조해 의미와 구조를 확인합니다.
5. 통과한 결과만 `polished_ko`에 새로 씁니다.

### Strict

다음 중 하나면 정밀 검증으로 올립니다.

- 8,000자를 넘는 긴 조각
- 표·각주·사료 인용이 많은 조각
- 수정률이 30%를 넘는 조각
- 의미 보존 또는 제목·각주·수치 대조가 한 번이라도 실패한 조각
- 사용자가 정밀 윤문을 요청한 경우

Strict에서는 초벌 번역과 윤문본을 별도로 대조하고, 실패한 부분만 되돌려 최대 두 차례 다시 윤문합니다. 해결되지 않은 항목은 억지로 고치지 않고 사람 검토 대상으로 남깁니다.

개별 조각이 Fast 검사를 통과했어도 책 전체는 Strict 방식으로 다시 검사합니다. 장 사이 용어와 문체, 제목·각주·표·캡션, TOC 계약과 실제 읽기 순서를 책 단위로 확인해야 합니다.

## 7. 병렬 윤문

- 기본적으로 서로 다른 작업 ID를 병렬로 처리합니다.
- 한 작업 ID의 `polished_ko`에는 한 작업자만 씁니다.
- 모든 작업자는 같은 용어집·스타일 표와 이 원칙을 읽습니다.
- coordinator는 남은 조각과 사용 가능한 메모리로 최대 4개의 추천 대상만 나열합니다. 현재 Claude/Codex 환경의 실제 동시 실행 한도를 적용하고 작업자를 시작하는 일은 실행 에이전트가 담당합니다.
- 용어집을 바꿔야 하면 해당 작업자는 임의로 확정하지 않고 충돌을 보고합니다. 합의된 변경을 공유한 뒤 나머지 조각에 적용합니다.

## 8. 과윤문 방지

- 수정률 30% 초과: 작업자가 경고하고 Strict 대조를 실행합니다. Python QC가 이 경고를 자동 승인했다는 뜻은 아닙니다.
- 수정률 50% 초과: Python QC가 결과 채택을 차단합니다. 파일을 자동 삭제하거나 덮어쓰지는 않으며, 작업자가 초벌 번역으로 돌아가 실패한 부분을 다시 윤문합니다.
- 한 문단을 통째로 다시 쓰기보다 문제가 확인된 문장과 구절만 고칩니다.
- 더 흔한 표현이라는 이유로 저자의 일관된 어휘를 지우지 않습니다.
- 문체를 매끄럽게 하려고 논지, 평가, 인과관계, 확신도를 바꾸지 않습니다.

## 9. 윤문 완료 검사

각 조각은 다음 항목을 모두 통과해야 합니다.

1. 제목의 텍스트·수준·순서가 초벌 번역과 일치함
2. 고유명사·수치·날짜·단위·직접 인용이 보존됨
3. 각주 번호와 표 행·열 수가 보존됨
4. 문단이나 문장이 누락·추가되지 않음
5. 원문의 단정·추정·유보 수준이 유지됨
6. 내부 작업 제목, PDF 페이지 표식, 반복 AI 고지가 없음
7. 번역투와 AI 문체를 줄였지만 장르와 저자 레지스터가 유지됨
8. 수정률이 허용 범위 안이며 미해결 항목이 보고됨

검사 실패 시 `polished_ko`를 완성으로 표시하지 않습니다. 기존 정상 파일을 덮어쓰지 말고 실패 원인과 다음 조치를 보고합니다.

### 자동 검사와 판단 검사의 경계

| Python이 자동 대조 | 에이전트 또는 사람이 판단 |
|---|---|
| 제목 순서, 수치·단위 순서, 통화, 각주 표식, URL·링크 대상 | 고유명사와 전문용어의 의미 일치 |
| 직접 인용, 블록 인용, 표의 행·열 형태 | 논증·인과관계·헤징과 저자 레지스터 |
| 50% 초과 수정률 차단 | 30% 초과 수정의 Strict 검토, 문장·내용 누락 여부 |

자동 검사는 의미 감사의 대체물이 아닙니다. 통과 결과도 원문과 초벌 번역을 대조하는 책 단위 Strict 검토를 거쳐야 합니다.

완료 보고에는 요청한 플랫폼·모델·effort, 적용한 윤문 원칙 버전, 변경률, 보류 항목 수를 남깁니다. 이 정보는 실행 환경의 자기 보고이며 Python 하네스가 실제 모델 신원을 검증했다는 뜻이 아닙니다.
